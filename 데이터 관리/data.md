## 1. 학습 데이터와 테스트 데이터의 분리

### 왜 분리하는가?

모델은 주어진 데이터의 패턴을 학습합니다.
그런데 학습에 사용한 데이터로 성능을 평가하면, 모델이 정답을 **"외웠는지"** 아니면 진짜 **"이해했는지"** 구분할 수 없습니다.

> 시험 문제를 미리 알고 본 시험에서 100점을 받는 것과 같습니다.

따라서 데이터를 반드시 **학습용(Training Set)**과 **평가용(Test Set)**으로 나눠야 합니다.

### 일반적인 분리 비율

| 구분 | 비율 | 용도 |
|------|------|------|
| **Training Set** (학습) | 70~80% | 모델이 패턴을 학습 |
| **Validation Set** (검증) | 10~15% | 하이퍼파라미터 조정, 과적합 확인 |
| **Test Set** (테스트) | 10~15% | 최종 성능 평가 (단 한 번만 사용) |

### 대표적인 데이터 분리 기법

데이터를 어떻게 분리하느냐에 따라 모델의 정확도와 신뢰성이 크게 달라집니다.
분리를 잘못하면 실제 성능보다 과대/과소 평가되어, 배포 후 예상치 못한 성능 저하가 발생합니다.

#### 1) Hold-Out (단순 분리)

가장 기본적인 방법으로, 데이터를 한 번만 학습/테스트로 나눕니다.

| 항목 | 내용 |
|------|------|
| **방법** | 데이터를 무작위로 섞은 뒤 비율에 따라 한 번 분리 |
| **장점** | 간단하고 빠름 |
| **단점** | 어떤 데이터가 테스트에 빠지느냐에 따라 결과가 달라짐 (분산이 큼) |
| **정확도 영향** | 데이터가 충분히 많으면(10만+) 안정적이지만, 적으면 운에 따라 정확도가 ±5~10% 흔들릴 수 있음 |

#### 2) K-Fold 교차 검증 (Cross Validation)

데이터를 K개의 묶음(Fold)으로 나누고, 매번 다른 묶음을 테스트로 사용하여 K번 학습/평가를 반복합니다.

| 항목 | 내용 |
|------|------|
| **방법** | 데이터를 K등분 → 1개를 테스트, 나머지 K-1개로 학습 → K번 반복 → 평균 성능 |
| **장점** | 모든 데이터가 한 번씩 테스트에 사용되어 평가가 안정적 |
| **단점** | K번 학습해야 하므로 시간이 K배 소요 |
| **정확도 영향** | Hold-Out 대비 정확도 편차가 크게 줄어듦. 일반적으로 K=5 또는 K=10 사용 |

> 예) 5-Fold CV 결과: 92%, 89%, 91%, 90%, 93% → 평균 **91%** (±1.4%)
> Hold-Out 한 번 결과: 89% 또는 93% — 어떤 게 진짜 성능인지 알 수 없음

#### 3) Stratified K-Fold (층화 교차 검증)

K-Fold에 **클래스 비율 유지**를 추가한 기법입니다. 분류 문제에서 가장 권장됩니다.

| 항목 | 내용 |
|------|------|
| **방법** | K-Fold와 동일하되, 각 Fold에서 클래스 비율이 원본과 동일하게 유지 |
| **장점** | 클래스 불균형 데이터에서도 안정적인 평가 가능 |
| **단점** | K-Fold와 동일 (시간 K배) |
| **정확도 영향** | 불균형 데이터(예: 양성 5%, 음성 95%)에서 일반 K-Fold 대비 정확도 편차가 2~5% 감소 |

> 예) 암 진단 데이터(양성 3%)에서 일반 K-Fold는 Fold마다 양성 비율이 0~8%로 흔들림
> → Stratified K-Fold는 모든 Fold에서 양성 3% 유지 → 훨씬 일관된 평가

#### 4) Time Series Split (시계열 분리)

시간 순서가 있는 데이터(주가, 날씨, 로그 등)에서 사용합니다. **미래 데이터로 과거를 예측하는 누수**를 방지합니다.

| 항목 | 내용 |
|------|------|
| **방법** | 시간 순서대로 정렬 → 과거 데이터로 학습, 미래 데이터로 테스트 (셔플 금지) |
| **장점** | 현실적인 예측 시나리오를 반영 |
| **단점** | 초기 데이터는 학습에만 사용되어 테스트 기회가 없음 |
| **정확도 영향** | 일반 Hold-Out을 시계열에 적용하면 정확도가 5~20% 과대평가될 수 있음. 이 기법으로 현실적인 성능 측정 가능 |

### 분리 기법에 따른 정확도 비교 (예시)

동일한 모델과 데이터에서 분리 방식만 바꿨을 때의 차이:

| 분리 기법 | 보고 정확도 | 실제 배포 정확도 | 신뢰도 |
|-----------|:-----------:|:---------------:|:------:|
| Hold-Out (1회) | 93% | 88% | 낮음 |
| K-Fold (K=5) | 91% ±1.4% | 90% | 높음 |
| Stratified K-Fold | 91% ±0.8% | 90% | 매우 높음 |
| 셔플 없이 순서대로 분리 | 95% | 78% | 매우 낮음 (데이터 누수 가능) |

> **핵심**: 분리를 대충 하면 "학습 시 93%인데 실제로는 78%"가 됩니다.
> 올바른 분리 기법을 선택하면 **보고 정확도와 실제 정확도의 격차**를 최소화할 수 있습니다.

### 분리 시 반드시 지켜야 할 원칙

- **데이터 누수(Data Leakage) 방지** — 테스트 데이터의 정보가 학습 과정에 절대 섞여 들어가면 안 됩니다. 누수가 발생하면 정확도가 비정상적으로 높게 나오고, 배포 후 급격히 떨어집니다.
- **셔플 후 분리** — 데이터가 특정 순서(날짜, 카테고리 등)로 정렬되어 있으면 편향이 생깁니다. 단, 시계열 데이터는 셔플하면 안 됩니다.
- **클래스 비율 유지** — 분류 문제에서는 반드시 Stratified 방식을 사용하여 각 클래스 비율을 동일하게 유지합니다.

## 2. 오버피팅(과적합) 문제

### 오버피팅이란?

모델이 학습 데이터에 **지나치게 맞춰져서**, 학습 데이터에서는 정확도가 높지만 새로운 데이터에서는 성능이 급격히 떨어지는 현상입니다.
모델이 데이터의 일반적인 패턴이 아니라 **노이즈(잡음)까지 외워버린 상태**입니다.

### 오버피팅 vs 언더피팅

| 상태 | 학습 정확도 | 테스트 정확도 | 설명 |
|------|:-----------:|:------------:|------|
| **언더피팅** | 낮음 | 낮음 | 모델이 너무 단순, 패턴 포착 실패 |
| **적절한 학습** | 높음 | 높음 | 일반적 패턴 포착, 새 데이터에도 잘 작동 |
| **오버피팅** | 매우 높음 | 낮음 | 노이즈까지 외워버린 상태 |

### 오버피팅 판별 방법

1. **학습 손실**(Training Loss)은 계속 줄어드는데 **검증 손실**(Validation Loss)이 다시 올라가기 시작 → 오버피팅 신호
2. 학습 정확도와 검증 정확도의 **격차가 점점 벌어짐** → 대표적인 징후

### 오버피팅 방지 기법

| 기법 | 설명 |
|------|------|
| **더 많은 데이터 확보** | 가장 근본적인 해결책 |
| **정규화 (Regularization)** | L1, L2 정규화로 가중치가 지나치게 커지는 것 방지 |
| **드롭아웃 (Dropout)** | 학습 중 뉴런 일부를 무작위 비활성화 |
| **조기 종료 (Early Stopping)** | 검증 손실이 줄어들지 않으면 학습 중단 |
| **데이터 증강 (Data Augmentation)** | 이미지 회전/반전/크기 조절로 데이터 확장 |
| **교차 검증 (Cross Validation)** | K-Fold로 여러 번 반복하여 일반화 능력 확인 |

## 3. 데이터 타입별 변환 방법

### 3-1. 텍스트 데이터 → CSV / Pandas DataFrame

텍스트 데이터는 **CSV** 파일로 저장하고, **Pandas**로 DataFrame으로 변환하여 다룹니다.

#### CSV 파일 예시

```csv
text,label
"오늘 날씨가 좋습니다",긍정
"서비스가 너무 불편합니다",부정
"보통이에요",중립
```

#### Pandas로 읽기

```python
import pandas as pd

df = pd.read_csv("data.csv")

print(df.head())       # 상위 5개 행 확인
print(df.shape)        # (행 수, 열 수)
print(df.info())       # 데이터 타입과 결측치
print(df.describe())   # 기본 통계 정보
```

#### DataFrame 핵심 기능

```python
df["text"]                             # 특정 열 선택
df.dropna()                            # 결측치 행 제거
df["label"].value_counts()             # 클래스별 데이터 수
df.sample(frac=1)                      # 데이터 셔플
df.to_csv("output.csv", index=False)   # CSV로 저장
```

### 3-2. 이미지 데이터 → RGB 값 변환

컴퓨터는 이미지를 **픽셀 단위의 숫자 배열**로 인식합니다.
각 픽셀은 **RGB**(Red, Green, Blue) 세 채널의 값(`0~255`)으로 구성됩니다.

#### 이미지 구조

| 타입 | 배열 형태 | 설명 |
|------|-----------|------|
| **흑백** (Grayscale) | `(높이, 너비)` | 각 픽셀: `0`(검정) ~ `255`(흰색) |
| **컬러** (RGB) | `(높이, 너비, 3)` | 각 픽셀: `[R, G, B]` 예) `[255, 0, 0]` = 빨간색 |

#### 이미지 → 숫자 배열 변환

```python
from PIL import Image
import numpy as np

img = Image.open("photo.jpg")
img_array = np.array(img)

print(img_array.shape)    # 예: (224, 224, 3)
print(img_array.dtype)    # uint8 (0~255)
print(img_array[0][0])    # 첫 픽셀의 [R, G, B]
```

#### 정규화 (Normalization)

픽셀 값을 `0~1` 범위로 변환하면 학습 속도와 안정성이 향상됩니다.

```python
img_normalized = img_array / 255.0    # 0~255 → 0.0~1.0
```

#### 이미지 크기 통일 (Resize)

모델 입력을 위해 모든 이미지 크기를 동일하게 맞춥니다.

```python
img_resized = img.resize((224, 224))
img_array = np.array(img_resized) / 255.0
```

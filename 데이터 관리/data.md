## 1. 학습 데이터와 테스트 데이터의 분리

### 왜 분리하는가?

모델은 주어진 데이터의 패턴을 학습합니다.
그런데 학습에 사용한 데이터로 성능을 평가하면, 모델이 정답을 **"외웠는지"** 아니면 진짜 **"이해했는지"** 구분할 수 없습니다.

> 시험 문제를 미리 알고 본 시험에서 100점을 받는 것과 같습니다.

따라서 데이터를 반드시 **학습용(Training Set)**과 **평가용(Test Set)**으로 나눠야 합니다.

### 일반적인 분리 비율

| 구분 | 비율 | 용도 |
|------|------|------|
| **Training Set** (학습) | 70~80% | 모델이 패턴을 학습 |
| **Validation Set** (검증) | 10~15% | 하이퍼파라미터 조정, 과적합 확인 |
| **Test Set** (테스트) | 10~15% | 최종 성능 평가 (단 한 번만 사용) |

### 분리 시 주의사항

- **무작위 셔플(Shuffle)**: 데이터가 특정 순서로 정렬되어 있으면 편향이 생깁니다. 반드시 섞은 뒤 분리합니다.
- **층화 샘플링(Stratified Split)**: 분류 문제에서 각 클래스의 비율이 학습/테스트 세트에 동일하게 유지되도록 나눕니다.
  - 예) 고양이 30%, 개 70%인 데이터 → 학습/테스트 세트 모두 이 비율 유지
- **데이터 누수(Data Leakage) 방지**: 테스트 데이터의 정보가 학습 과정에 절대 섞여 들어가면 안 됩니다.

## 2. 오버피팅(과적합) 문제

### 오버피팅이란?

모델이 학습 데이터에 **지나치게 맞춰져서**, 학습 데이터에서는 정확도가 높지만 새로운 데이터(테스트 데이터)에서는 성능이 급격히 떨어지는 현상입니다.
즉, 모델이 데이터의 일반적인 패턴이 아니라 **노이즈(잡음)까지 외워버린 상태**입니다.

### 오버피팅 vs 언더피팅

| 상태 | 학습 정확도 | 테스트 정확도 | 설명 |
|------|:-----------:|:------------:|------|
| **언더피팅** (Underfitting) | 낮음 | 낮음 | 모델이 너무 단순해서 패턴조차 포착 못함 |
| **적절한 학습** (Good Fit) | 높음 | 높음 | 일반적인 패턴을 포착, 새 데이터에도 잘 작동 |
| **오버피팅** (Overfitting) | 매우 높음 | 낮음 | 노이즈까지 외워버린 상태 |

### 오버피팅 판별 방법

1. **학습 손실**(Training Loss)은 계속 줄어드는데, **검증 손실**(Validation Loss)이 다시 올라가기 시작 → 오버피팅 신호
2. 학습 정확도와 검증 정확도의 **격차가 점점 벌어짐** → 대표적인 징후

## 3. 데이터 타입별 변환 방법

### 3-1. 텍스트 데이터 → CSV / Pandas DataFrame

텍스트 데이터는 **CSV**(Comma-Separated Values) 파일로 저장하고, Python의 **Pandas** 라이브러리를 통해 DataFrame으로 변환하여 다룹니다.

#### CSV 파일이란?

각 행이 하나의 데이터 샘플이고, 열은 쉼표(`,`)로 구분된 텍스트 파일입니다.

```csv
text,label
"오늘 날씨가 좋습니다",긍정
"서비스가 너무 불편합니다",부정
"보통이에요",중립
```

#### Pandas DataFrame으로 읽기

```python
import pandas as pd

df = pd.read_csv("data.csv")

print(df.head())       # 상위 5개 행 확인
print(df.shape)        # (행 수, 열 수) 확인
print(df.info())       # 각 열의 데이터 타입과 결측치 확인
print(df.describe())   # 기본 통계 정보
```

#### DataFrame 핵심 기능

```python
df["text"]                          # 특정 열 선택
df.dropna()                         # 결측치가 있는 행 제거
df["label"].value_counts()          # 각 클래스별 데이터 수 확인
df.sample(frac=1)                   # 데이터 셔플
df.to_csv("output.csv", index=False)  # 다시 CSV로 저장
```

### 3-2. 이미지 데이터 → RGB 값 변환

컴퓨터는 이미지를 **픽셀 단위의 숫자 배열**로 인식합니다.
각 픽셀은 **RGB**(Red, Green, Blue) 세 가지 색상 채널의 값(`0~255`)으로 구성됩니다.

#### 이미지의 구조

| 타입 | 배열 형태 | 설명 |
|------|-----------|------|
| **흑백** (Grayscale) | `(높이, 너비)` 2차원 | 각 픽셀: `0`(검정) ~ `255`(흰색) |
| **컬러** (RGB) | `(높이, 너비, 3)` 3차원 | 각 픽셀: `[R, G, B]` 예) `[255, 0, 0]` = 빨간색 |

#### 이미지를 숫자 배열로 변환하기

```python
from PIL import Image
import numpy as np

img = Image.open("photo.jpg")       # 이미지 파일 열기
img_array = np.array(img)           # NumPy 배열로 변환

print(img_array.shape)              # 예: (224, 224, 3) → 224x224 크기, RGB 3채널
print(img_array.dtype)              # uint8 (0~255 범위의 정수)
print(img_array[0][0])              # 첫 번째 픽셀의 [R, G, B] 값
```

#### 정규화 (Normalization)

모델에 입력하기 전에 픽셀 값을 `0~1` 범위로 정규화하는 것이 일반적입니다.
학습 속도와 안정성이 향상됩니다.

```python
img_normalized = img_array / 255.0  # 0~255 → 0.0~1.0
print(img_normalized[0][0])         # 예: [0.678, 0.432, 0.231]
```

#### 이미지 크기 통일 (Resize)

모델에 입력하려면 모든 이미지의 크기가 동일해야 합니다.
일반적으로 `224x224` 또는 `256x256` 크기로 맞춥니다.

```python
img_resized = img.resize((224, 224))
img_array = np.array(img_resized) / 255.0
```
